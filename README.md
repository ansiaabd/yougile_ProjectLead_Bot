# 🤖 @My_ProjectLead_Bot — Управление задачами через Telegram + Yougile

Telegram-бот для управления задачами с двусторонней синхронизацией с [Yougile](https://ru.yougile.com).  
Роли (админ, модератор, пользователь), подтверждение выполнения, вложение файлов/комментариев, уведомления.

---

## 🔄 Интеграция с Yougile

### Telegram → Yougile

| Действие в боте | Результат в Yougile |
|---|---|
| `/add` → выбор проекта → исполнитель → название/описание/дедлайн | Задача создаётся на доске **«Задачи»** |
| `/take <id>` | Задача перемещается на доску **«В работе»** |
| `/done <id>` + комментарий/файл | Задача → **«На проверке»**, добавляется комментарий исполнителя |
| ✅ Подтвердить (админ/модератор) | Задача → **«Готово»**, `completed = true` |
| ❌ Отклонить (админ/модератор) | Задача → **«В работе»**, добавляется комментарий |

### Yougile → Telegram

При изменениях в Yougile (создание, перемещение, переименование, выполнение задачи) — бот присылает уведомление исполнителю, создателю задачи и администратору.

---

## ✨ Возможности

- **Yougile-синхронизация** — задачи, статусы, комментарии в обе стороны
- **Выбор проекта** — при создании задачи бот показывает список проектов из Yougile
- **Ролевая модель** — админ, модератор, пользователь
- **Создание задач** — через `/add` (пошагово с выбором проекта, исполнителя) или одной строкой
- **Умный парсинг дат** — «11 июня в 14», «завтра в 15», «сегодня в 20», «08.06 18:00»
- **Комментарий и файл к выполнению** — после отметки «выполнено» бот предлагает добавить 💬 комментарий или 📎 файл
- **Подтверждение выполнения** — исполнитель → создатель задачи или админ подтверждает (✅/❌)
- **Уведомления о просрочках** — каждую минуту проверяет дедлайны
- **Inline-кнопки** — ✅ Выполнено, 🗑 Удалить, ✅ Подтвердить, ❌ Отклонить
- **Natural language** — «выполнено», «сделано», «готово» отмечают активную задачу

## 📋 Команды

### Для всех
| Команда | Описание |
|---------|----------|
| `/add` | Создать задачу (проект → исполнитель → название → описание → дедлайн) |
| `/take <id>` | Взять задачу в работу |
| `/done <id>` | Отметить выполненной + добавить комментарий/файл |
| `/delete <id>` | Удалить задачу |
| `/list` | Мои задачи |
| `/overdue` | Просроченные задачи |
| `/menu` | Быстрое меню |
| `/cancel` | Отменить создание |
| `/help` | Справка |
| «выполнено» / «сделано» / «готово» | Быстрая отметка |

### Админ и модератор
| Команда | Описание |
|---------|----------|
| `/pending` | Задачи на подтверждении |

### Админ
| Команда | Описание |
|---------|----------|
| `/users` | Список пользователей с ролями |
| `/promote <id>` | Назначить модератором |
| `/demote <id>` | Снять модератора |
| `/removeuser <id>` | Удалить пользователя |
| `/setup_project` | Создать доски «Задачи», «В работе», «На проверке», «Готово» в проектах Yougile |
| `/setup_webhooks <url>` | Настроить вебхуки Yougile для обратной синхронизации |

## 🏗 Обязательная структура проекта в Yougile

В каждом проекте должны быть доски с названиями:

| Доска | Назначение |
|-------|-----------|
| **Задачи** | Новые задачи из бота |
| **В работе** | Исполнитель взял задачу |
| **На проверке** | Ожидает подтверждения |
| **Готово** | Выполнено |

Создаются автоматически командой `/setup_project`.

## 🚀 Быстрый старт

### 1. Установка

```bash
git clone https://github.com/ansiaabd/yougile_ProjectLead_Bot.git
cd yougile_ProjectLead_Bot
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 2. Настройка

Создать `.env` в корне проекта:

```env
BOT_TOKEN=токен_от_@BotFather
ADMIN_ID=ваш_telegram_id

# Yougile (https://ru.yougile.com)
YOUGILE_COMPANY_ID=id_компании
YOUGILE_API_KEY=api_ключ
```

Получить `COMPANY_ID` и `API_KEY`: **Yougile → Настройки компании → API**.

### 3. Запуск

```bash
python main.py
```

### 4. Настройка интеграции (один раз)

В Telegram отправьте боту:

```
/setup_project
```

Доски будут созданы во всех проектах Yougile.

### 5. Вебхуки (Yougile → Telegram)

Для обратной синхронизации нужен публичный URL.  
С помощью [ngrok](https://ngrok.com):

```bash
ngrok http 8787
```

Скопировать HTTPS-ссылку и отправить боту:

```
/setup_webhooks https://ваш-id.ngrok-free.app
```

## 📁 Структура проекта

```
├── main.py              # Точка входа (бот + webhook-сервер)
├── config.py            # .env конфиг
├── bot/
│   ├── handlers.py      # Все обработчики команд/кнопок
│   ├── messages.py      # Шаблоны сообщений
│   └── keyboards.py     # Inline-клавиатуры

│   ├── crud.py          # Операции с БД
│   └── __init__.py      # SQLite (WAL mode)

│   ├── client.py        # API-клиент Yougile (httpx)
│   ├── webhook.py       # HTTP-сервер для вебхуков Yougile
│   ├── config.py        # Чтение YOUGILE_* из .env
│   └── credentials.md   # Инструкция по получению ключей
├── scheduler/
│   └── tasks.py         # Проверка просрочек (60с)
├── utils/
│   └── date_parser.py   # Парсинг русских дат
├── docs/
│   ├── spec.md          # Техническое задание
│   └── guide.md         # Руководство пользователя
└── .env.example         # Пример конфига
```
