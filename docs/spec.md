# Техническое задание — TaskBot Calendar Sync

## 1. Назначение

Telegram-бот для управления задачами с автоматической синхронизацией в календарь ПК (Outlook / Google Calendar). Позволяет быстро создавать задачи, назначать исполнителей, отслеживать дедлайны и получать напоминания.

## 2. Функциональные требования

### 2.1. Управление задачами
| Команда | Описание |
|---------|----------|
| `/add <название> / <дедлайн> / <исполнитель>` | Создать задачу |
| `/list` | Показать все активные задачи |
| `/list all` | Показать все задачи (включая выполненные) |
| `/done <id>` | Отметить задачу как выполненную |
| `/delete <id>` | Удалить задачу |
| `/edit <id> <поле> <значение>` | Изменить поле задачи |
| `/help` | Показать справку |

### 2.2. Система оповещений
- Проверка просроченных задач каждую минуту (APScheduler).
- Уведомление исполнителю при просрочке.
- Уведомление администратору о просрочке.

### 2.3. Интеграция с календарём
- При создании задачи — создание события в календаре (Outlook или Google Calendar).
- Дата/время события = дедлайн задачи.
- При отметке `/done` — пометка события как выполненного.
- При изменении дедлайна — обновление события.
- При удалении задачи — удаление события.

### 2.4. UI/UX
- Inline-кнопки для быстрых действий (выполнить, перенести дедлайн).
- Логирование всех действий в файл.
- Обработка ошибок ввода (неверный формат даты, потеря связи).

## 3. Поля задачи (модель данных)

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| `id` | INTEGER PK | да | Автоинкремент |
| `title` | TEXT | да | Название задачи |
| `description` | TEXT | нет | Описание |
| `assignee` | TEXT | да | Исполнитель (username Telegram) |
| `deadline` | DATETIME | да | Дедлайн |
| `status` | TEXT | да | `active` / `done` / `overdue` |
| `created_at` | DATETIME | да | Дата создания |
| `calendar_event_id` | TEXT | нет | ID события в календаре |

## 4. Пользовательские роли

- **Администратор** — владелец бота, получает уведомления о просрочках, может управлять любыми задачами.
- **Исполнитель** — пользователь, назначенный на задачу, получает напоминания.

## 5. Ограничения

- Бот работает в однопользовательском режиме (один администратор) или в режиме малой группы.
- Календарь привязан к ПК администратора.

# Архитектура

## Стек технологий

| Компонент | Технология | Обоснование |
|-----------|-----------|-------------|
| Язык | Python 3.9+ | Стабильность, широкая поддержка библиотек |
| Telegram API | python-telegram-bot 20+ | Асинхронный, современный, поддерживает inline-кнопки |
| База данных | SQLite | Не требует сервера, встроенная, достаточно для малой группы |
| Планировщик | APScheduler | Лёгкий, in-process, проверка дедлайнов |
| Календарь (Google) | google-api-python-client | Официальная библиотека, OAuth2 |
| Календарь (Outlook) | pywin32 (COM) | Только Windows, прямой доступ к Outlook |

## Варианты интеграции с календарём

### Outlook (Windows)
- Используется `pywin32` для работы с COM-объектом Outlook.
- Создание встречи через `CreateItem(1)`.
- Требует установленного Outlook на ПК.

### Google Calendar (кроссплатформенно)
- OAuth2 через `google-auth-oauthlib`.
- API v3 через `google-api-python-client`.
- Требует проект в Google Cloud Console с включённым Calendar API.

## Обработка ошибок

- Неверный формат даты: ответ пользователю с примером корректного формата.
- Потеря связи с Telegram: повтор при следующем подключении.
- Ошибка календаря: логирование, задача сохраняется в БД без события.
